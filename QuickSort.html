Quick Sort¶

QuickSort adalah algoritma pengurutan berdasarkan prinsip Bagi dan Taklukkan yang memilih sebuah elemen sebagai pivot dan mempartisi array yang diberikan di sekitar pivot yang dipilih dengan menempatkan pivot pada posisi yang benar dalam array yang telah diurutkan.

Pada dasarnya ada tiga langkah dalam algoritma tersebut:

  • Pilih Pivot: Pilih salah satu elemen dari array sebagai pivot. Pilihan pivot dapat bervariasi (misalnya, elemen pertama, elemen terakhir, elemen acak, atau median).

  • Mempartisi Larik: Susun ulang larik di sekitar elemen pivot. Setelah dipartisi, semua elemen yang lebih kecil dari pivot akan berada di sebelah kirinya, dan semua elemen yang lebih besar dari pivot akan berada di sebelah kanannya.

  • Panggilan Rekursif: Terapkan proses yang sama secara rekursif pada dua sub-array yang telah dipartisi.

  • Base Case: Rekursi berhenti ketika hanya tersisa satu elemen dalam sub-array, karena satu elemen sudah terurut.

Algoritma Partition¶

Proses kunci dalam quickSort adalah partition(). Ada tiga algoritma umum untuk partition. Semua algoritma ini memiliki kompleksitas waktu O(n).

  • Partition Sederhana : Di sini kita membuat salinan array. Pertama-tama kita masukkan semua elemen yang lebih kecil, lalu semua elemen yang lebih besar. Terakhir, kita salin array sementara kembali ke array asli. Ini membutuhkan ruang tambahan O(n).

  • Partition Lomuto : Kami menggunakan partition ini dalam artikel ini. Ini adalah algoritma sederhana, kita melacak indeks elemen yang lebih kecil dan terus melakukan pertukaran. Kami menggunakannya di sini dalam artikel ini karena kesederhanaannya.

  • Partition Hoare : Ini adalah yang tercepat dari semuanya. Di sini kita menelusuri array dari kedua sisi dan terus menukar elemen yang lebih besar di sebelah kiri dengan elemen yang lebih kecil di sebelah kanan selama array belum dipartisi. Silakan lihat perbandingan Hoare vs Lomuto untuk detailnya.

Logikanya sederhana, kita mulai dari elemen paling kiri dan melacak indeks elemen yang lebih kecil (atau sama) sebagai i . Saat menelusuri, jika kita menemukan elemen yang lebih kecil, kita menukar elemen saat ini dengan arr[i] . Jika tidak, kita mengabaikan elemen saat ini.

def partition(array, low, high):
    pivot = array[high]
    i = low - 1

    for j in range(low, high):
        if array[j] <= pivot:
            i += 1
            array[i], array[j] = array[j], array[i]

    array[i+1], array[high] = array[high], array[i+1]
    return i+1

def quicksort(array, low=0, high=None):
    if high is None:
        high = len(array) - 1

    if low < high:
        pivot_index = partition(array, low, high)
        quicksort(array, low, pivot_index-1)
        quicksort(array, pivot_index+1, high)

# partition function
def partition(arr, low, high):
    
    # choose the pivot
    pivot = arr[high]
    
    # index of smaller element and indicates 
    # the right position of pivot found so far
    i = low - 1
    
    # traverse arr[low..high] and move all smaller
    # elements to the left side. Elements from low to 
    # i are smaller after every iteration
    for j in range(low, high):
        if arr[j] < pivot:
            i += 1
            swap(arr, i, j)
    
    # move pivot after smaller elements and
    # return its position
    swap(arr, i + 1, high)
    return i + 1

# swap function
def swap(arr, i, j):
    arr[i], arr[j] = arr[j], arr[i]

# the QuickSort function implementation
def quickSort(arr, low, high):
    if low < high:
        
        # pi is the partition return index of pivot
        pi = partition(arr, low, high)
        
        # recursion calls for smaller elements
        # and greater or equals elements
        quickSort(arr, low, pi - 1)
        quickSort(arr, pi + 1, high)

Analisis Kompleksitas Quick Sort¶

Kompleksitas Waktu:

  • Kasus Terbaik: (Ω(n log n)), Terjadi ketika elemen pivot membagi array menjadi dua bagian yang sama.

  • Kasus Rata-Rata (θ(n log n)), Secara rata-rata, pivot membagi array menjadi dua bagian, tetapi tidak selalu sama.

  • Kasus Terburuk: (O(n²)), Terjadi ketika elemen terkecil atau terbesar selalu dipilih sebagai pivot (misalnya, array yang sudah diurutkan).

Ruang Tambahan:

  • Skenario terburuk: O(n) karena partisi yang tidak seimbang menyebabkan pohon rekursi miring yang membutuhkan tumpukan panggilan berukuran O(n).

  • Skenario terbaik: O(log n) sebagai hasil dari partisi seimbang yang menghasilkan pohon rekursi seimbang dengan tumpukan panggilan berukuran O(log n).

Keunggulan Quick Sort¶

  • Ini adalah algoritma bagi-dan-taklukkan yang mempermudah penyelesaian masalah.

  • Metode ini efisien pada kumpulan data yang besar.

  • Sistem ini memiliki biaya operasional yang rendah, karena hanya membutuhkan sedikit memori untuk berfungsi.

  • Metode ini ramah terhadap cache karena kita bekerja pada array yang sama untuk pengurutan dan tidak menyalin data ke array bantu mana pun.

  • Algoritma serbaguna tercepat untuk data besar ketika stabilitas tidak diperlukan.

  • Algoritma ini bersifat rekursif ekor , sehingga semua optimasi panggilan ekor dapat dilakukan.

Kekurangan dari Quick Sort¶

  • Kompleksitas waktu terburuknya adalah O(n² ) , yang terjadi ketika pivot dipilih dengan buruk.

  • Ini bukan pilihan yang baik untuk kumpulan data kecil.

  • Ini bukan pengurutan stabil, artinya jika dua elemen memiliki kunci yang sama, urutan relatifnya tidak akan dipertahankan dalam output yang

  • diurutkan dalam kasus pengurutan cepat, karena di sini kita menukar elemen sesuai dengan posisi pivot (tanpa mempertimbangkan posisi aslinya).

Aplikasi Quick Sort¶

  • Mengurutkan dataset besar secara efisien di dalam memori.
  • Digunakan dalam fungsi pengurutan pustaka (seperti C++ std::sort dan Java Arrays.sort untuk tipe data primitif).
  • Mengatur catatan dalam basis data untuk pencarian yang lebih cepat.
  • Langkah pra-pemrosesan dalam algoritma yang membutuhkan input yang sudah diurutkan (misalnya, pencarian biner, teknik dua-pointer).
  • Mencari elemen terkecil/terbesar ke-k menggunakan Quickselect (varian dari quicksort).
  • Mengurutkan array objek berdasarkan beberapa kunci (komparator kustom).
  • Algoritma kompresi data (seperti pra-pemrosesan pengkodean Huffman).
  • Grafik dan geometri komputasional (misalnya, algoritma convex hull).
